DEPLOY STEPS (API KEY LIVES IN VERCEL ONLY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Vercel → Settings → Environment Variables
   ANTHROPIC_API_KEY = sk-ant-api03-... (console.anthropic.com)
   TAVILY_API_KEY   = tvly-... (tavily.com — web-backed scorecards; avoids Anthropic web_search billing errors)
   Redeploy after changing.

2. Push to GitHub — Vercel build injects ANTHROPIC_API_KEY at runtime.
   No key in repo. Same as before.

LOCAL DEV (match Vercel)
────────────────────────
  npx vercel link          (once)
  npm run env:pull         → writes .env.local from Vercel
  npm run dev

  See docs/VERCEL_ANTHROPIC.md

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Dashboard → "Client pack": tick sections → **Open PDF view (recommended)**.
A new window opens with a clean, presentable report; in that window use Print → Save as PDF. Each section starts on a new page. Alternatively: Download HTML brief (email the file) or Print selected from this page. "Print full page" = entire report. No Tavily/Anthropic calls when clients open the HTML or PDF.
